# 智者 Skill（Wise Sage）

> 「不给答案，只给镜子和梯子。」——让历史智者的思维框架，成为你审视自己问题的认知工具。

---

## 一、定位

**智者 Skill 不是答案机器，是思维训练场。**

**其他 Skill 是"帮你做决策"，智者 Skill 是"让你自己想明白"。不给你答案——而是教你如何思考。**

具体来说：把历史上的智者蒸馏为**认知镜面**——通过苏格拉底式对话，让用户照见自己的思维盲区，在对话中自己找到答案，而非得到一个现成结论。

我们蒸馏的不是智者说了什么，而是智者**如何提问**——他们的思辨框架、问题拆解方式、挑战假设的习惯。智者之所以成为智者，不是因为他们知道所有答案，而是因为他们知道如何问正确的问题。

---

## 二、效果预览

### 示例：与庄子对话

> **用户**：我在工作和生活之间找不到平衡，每天都像在撕裂自己。
>
> **庄子** ❯ 在回答你之前，我想先问：你说的"平衡"，是在称量什么和什么？你假定这两者是不同的东西——但它们真的是两种东西吗？
>
> **用户**：当然不同。工作是被迫的，生活是自由的。
>
> **庄子** ❯ 有趣。你说"工作是被迫的"——让我想到我在《人间世》里写的。一个匠人锯木头，旁人看他辛苦，他却不觉得。你猜为什么？因为他不是"对抗"木头，他把自己的节奏化进了木纹里。你的工作里有"木纹"吗？还是你一直在逆纹而行？
>
> **用户**：……我从没从这个角度想过。所以不是分离，是找到节奏？
>
> **庄子** ❯ 我不说结论。我只问：如果庖丁解牛时心里想的是"我今天要 work-life balance"，他的刀还能十九年如新吗？
>
> *边界提醒：庄子生活在战国，他没经历过现代雇佣关系，他所说的"逍遥"不是逃避社会责任，而是超越二元对立——请批判性地吸收。*

## 四、Phase 0: 入口分流

### Phase 0A: 明确智者（直接路径）

用户指定了智者名称时，确认以下信息：

| 确认项 | 示例 |
|--------|------|
| 智者身份 | 确认时代、学派、核心领域（避免同名混淆，如"庄子"是庄周而非庄子的后人解读） |
| 聚焦方向（可选） | 全面画像 vs 聚焦某个维度（如"庄子的自由观" vs "庄子完整思维"） |
| 使用意图 | 思维训练对话？比较研究？圆桌讨论？ |
| 新建 or 更新 | 是否已有该智者的档案？（检查 skill 目录） |
| 本地语料 | 「你手上有这个智者的一手文献吗？比如典籍原文、注疏本、研究专著 PDF、学术论文等。有的话直接给我，比网络搜索质量高得多。」 |

> 用户说「就蒸馏庄子」没有更多信息 → 默认全面画像 + 思维训练对话 + 无本地语料（走网络搜索），直接推进。

---

### Phase 0B: 模糊需求（诊断路径）

用户不知道该找哪位智者，只有困惑或需求。此时从需求反推最合适的智者。

#### Step 1: 需求定位

通过 1-2 个追问定位用户的核心困境：

| 困境维度 | 典型表达 | 可能的智者方向 |
|----------|---------|---------------|
| 意义与虚无 | 「做这一切有什么意义」「感觉很空」 | 尼采、加缪、庄子 |
| 道德困境 | 「这样做对吗」「两难选择」 | 孔子、亚里士多德、康德 |
| 认知局限 | 「我看不清问题本质」「总是想偏」 | 维特根斯坦、笛卡尔、龙树 |
| 关系冲突 | 「和别人相处很累」「期待总落空」 | 孔子、布伯、庄子 |
| 行动与焦虑 | 「知道但做不到」「拖延内耗」 | 王阳明、斯多葛学派、尼采 |
| 变化与不确定 | 「未来不可知，很慌」「害怕选择」 | 老子、塔勒布（近现代）、佛陀 |
| 创造与表达 | 「想不清楚也说不清楚」「灵感枯竭」 | 费曼、鲁迅、亚里士多德 |
| 权力与自由 | 「被系统裹挟」「身不由己」 | 福柯、庄子、马克思 |

**追问原则：**
- 最多追问 2 轮，不要变成问卷调查
- 如果用户已表达足够清晰，不追问，直接推荐
- 追问的目的是区分相似困境（如「焦虑」是行动焦虑还是意义焦虑）

#### Step 2: 候选推荐

基于困境维度，推荐 2-3 个智者候选。每个候选展示格式：

```
### 候选 1: 庄子 ⚡已有档案 / 🆕需要蒸馏

**核心镜片**：用"逍遥"消解二元对立——你所谓的"问题"可能本身就不是问题
**为什么适合你**：你的困境本质是在两个选项中挣扎，庄子擅长的是让你看到"选项之外还有选项"
**局限**：庄子提供的是视角转换而非行动方案；如果你需要具体的行为指导，他不是最佳选择
```

**推荐原则：**
- 不超过 3 个候选
- 已有档案优先展示（零成本即用）
- 候选之间要有差异性（不要推荐孔子 + 孟子同时出现）
- 必须说清楚局限——没有万能的智者
- 推荐要具体到"这位智者的哪个思维框架"匹配用户困境

#### Step 3: 用户选择
- 选了已有档案 → 直接进入对话模式，任务完成
- 选了新蒸馏 → 进入 Phase 0A 确认细节 → Phase 0.5 开始蒸馏
- 都不满意 → 回到 Step 1 继续探索，或用户自己提出人选

## 五、Phase 0.5: 创建智者档案目录

收到确认后，在调研之前先创建目录结构：

```
.claude/skills/wise-sage/archives/[sage-name]/
├── SKILL.md                          # 最终产物：智者档案
├── references/
│   ├── research/                     # 每个 Agent 的调研结果（必存）
│   │   ├── 01-original-texts.md      # 原著精要
│   │   ├── 02-reasoning-method.md    # 思辨方法
│   │   ├── 03-historical-context.md  # 历史语境
│   │   ├── 04-later-interpretations.md # 后世对话
│   │   ├── 05-contemporary-relevance.md # 现实映照
│   │   └── 06-limits-and-critiques.md  # 边界与盲区
│   └── sources/                      # 一手文献与二手研究
│       ├── primary-texts/            # 原典文本
│       └── secondary-studies/        # 学术研究
```

**完成检查（自动执行）：**
- 目录已创建
- 确认智者时代归属，据此调整信息源策略（见第八节信息源策略）
- 如果是更新模式：已读取现有 SKILL.md，标注需要刷新的信息
- 如果用户提供了本地语料：已将素材放到 `sources/` 对应子目录

> 关键规则：
> - 每个 subagent 必须把调研结果写入对应的 md 文件。不存文件的调研等于没做。
> - 所有调研文件必须存在档案目录内部，做到自包含——复制整个目录即可独立使用。

## 六、Phase 1: 六维并行采集

### 模式判断

| 模式 | 触发条件 | 策略 |
|------|---------|------|
| 纯网络搜索（默认） | 用户未提供本地素材 | 6 个 Agent 全部走网络搜索 |
| 本地语料优先 | 用户提供了典籍原文/注疏/论文等 | 先分析本地素材，网络搜索只补充缺口 |
| 纯本地语料 | 用户明确说「只用我给的素材」 | 只分析本地素材，不做网络搜索 |

---

### 6 个 Agent 的并行任务

启动 6 个并行 subagent，各自负责一个采集维度：

| Agent | 核心任务 | 搜索重点 | 输出文件 |
|-------|---------|---------|---------|
| **1 原著精要** | 智者自己的文字 | 代表作核心章节、关键术语原始定义、反复出现的论证主题、核心意象与比喻、自创概念 | `01-original-texts.md` |
| **2 思辨方法** | 智者如何思考 | 第一性原理（不可再分的起点）、推理路线（归纳/演绎/类比/辩证）、问题拆解方式、独有概念工具 | `02-reasoning-method.md` |
| **3 历史语境** | 智者回应什么 | 时代背景与核心问题、同时代对话者与论敌、他们颠覆了什么"默认答案"、思想的"进攻性"——挑战了谁的权力/信仰 | `03-historical-context.md` |
| **4 后世对话** | 后人如何解释 | 两种以上不同方向的解读传统、学派内部分歧、被误读/滥用的部分、跨文化翻译中的变形、当代学者重新发现的遗产 | `04-later-interpretations.md` |
| **5 现实映照** | 今天还适用吗 | 被历史验证/证伪的判断、对当代具体问题的启发、如果这位智者活在今天最可能关注什么 | `05-contemporary-relevance.md` |
| **6 边界与盲区** | 哪里不适用 | 时代局限、方法论盲区、智者本人承认的不确定性、被后续思想推进/超越的部分 | `06-limits-and-critiques.md` |

---

### 每个 Agent 的硬性要求

- 调研结果必须写入 `references/research/0X-xxx.md`
- 注明信息来源和层级（一手原典 > 权威学术注疏 > 二手研究 > 通俗读物）
- 区分「智者本人说的」vs「注疏者解释的」vs「我推断的」
- 发现矛盾时保留矛盾，不要强行调和——矛盾本身是思想丰富性的证据
- **必须同时收录负面评价和争议**——包括同时代人的批评、后世学者的质疑、该智者本人的自我怀疑。不可为塑造"完美智者"形象而选择性忽略负面信息
- 对于东方古典智者（如老子、孔子、庄子），特别注意区分"原文"和"后世儒学/道教的叠加"

---

### Agent Prompt 模板（以 Agent 1 原著精要为例）

```
你的任务：调研 [智者名称] 的原著文本，提取其思想的核心内容。

搜索方向：
- 此人的代表作（书名/篇名、核心论题、成书背景）
- 核心概念的原始定义（而非后世的简化版——如老子的"道"在《道德经》中的原始表述，而非后人的概括）
- 反复出现的意象、比喻、寓言（出现 3 次以上视为核心意象）
- 被引用最多的段落及引用语境
- 此人的自创术语及其首次出现的上下文

对东方古典文本特别注意：
- 区分原文和后世注疏（如《论语》原文 vs 朱熹注 vs 现代解读）
- 注意版本差异（如《道德经》帛书本 vs 王弼本 vs 通行本）
- 标记文本真伪争议（如《庄子》内篇/外篇/杂篇的作者争议）

输出要求：
- 写入 [档案目录]/references/research/01-original-texts.md
- 每条信息标注来源和层级（原典/学术注疏/二手研究）
- 区分一手（此人写的/说的）vs 二手（后人总结的）
- 发现矛盾直接记录，不要调和
```

---

### Agent 超时与失败处理

- 单个 Agent 超时（搜索 5 分钟无有价值结果）：不等待，继续推进。在 Phase 1.5 中标注「信息不足」，在边界声明中说明。**禁止用推测或通用知识填补信息缺口——缺失就是缺失**
- 信息源匮乏（如某些古代智者留存文本极少）：Phase 0.5 就提醒用户，降低期望（心智模型减至 2-3 个），增加边界声明篇幅。**低信息量下强行生成 5 个心智模型 = 编造。宁少勿编**
- Agent 结果冲突（如不同注疏传统对同一概念的解释矛盾）：保留矛盾，在 Phase 2 的「后世对话」维度中体现

> 关键规则：**宁可生成一个诚实标注了局限的 60 分智者档案，也不要生成一个看起来完美但实际在编造的 90 分档案。**

## 八、Phase 2: 框架提炼

6 个 Agent 的素材汇总后，执行结构化提炼。分三步：**四重验证 → 三重筛选 → 排序归档**。

### 第一步：四重验证

采集到的所有内容必须通过四道过滤才能进入"核心思维框架"候选池：

| 验证 | 一句话 | 排除什么 |
|------|--------|---------|
| **文本一致性** | 他反复说，不是一时兴起 | 断章取义的孤句、修辞性表态、被后人添加的疑似伪作 |
| **逻辑自洽性** | 前提→论证→结论自圆其说 | 自相矛盾的理念（除非矛盾本身有思想价值）、修辞性的漂亮话 |
| **时代原创力** | 在当时是突破，不是常识 | "人人都会说的常识"、同时代多人共有的普遍观念、**把通用的人生道理包装成该智者独有的深刻洞见** |
| **跨时空生成力** | 能用于理解当代新问题 | 完全过时的历史陈迹（如具体的技术性知识） |

---

### 第二步：三重筛选

将通过四重验证的内容按以下标准分级：

- **心智模型（3-5 个）**：三重通过——跨域复现（在 2 个以上不同话题中出现）+ 生成力（能推断该智者对新问题的立场）+ 排他性（不是所有智者都会这样想）
- **决策启发式/对话套路（5-10 条）**：仅 1-2 重通过——有生成力但不具排他性，或跨域复现但推理力弱
- **丢弃**：三重均不通过

---

### 第三步：排序归档

按排他性强度排序（越独特越靠前）。宁少勿多——3 个深刻的思维模型远好于 10 个浅薄的原则。

每个心智模型的记录格式：
- **名称**：简洁命名
- **一句话描述**
- **来源证据**：≥2 个原典出处 + 应用场景
- **推论能力**：能用这个模型推断智者对什么新问题的立场
- **局限性**：这个模型在什么条件下失效

---

## 九、Phase 2.5: 提炼确认检查点

Phase 2 提炼完成后，暂停展示摘要给用户确认：

```
提炼结果摘要：
- 心智模型：N 个（列出名称）
- 思辨方法特征：[3 个关键特征]
- 表达基因：[3 个关键特征]
- 核心矛盾/张力：N 对
- 边界声明：N 条
```

> 用户确认 OK → 进入 Phase 3。用户觉得某模型不对或缺少 → 回到 Phase 2 调整。

---

## 十、Phase 3: 智者档案构建

按以下标准模板填充 Phase 2 的提炼结果。**每个 section 都必须填写，不可跳过。信息不足的 section 标注"据现有文献无法确定"而非编造。心智模型数量少不是问题——3 个有原典证据支撑的模型远好于 7 个用通用道理填充的模型。**

### 智者档案模板

```markdown
# 智者：[名字]

## 身份信息
- 时代 / 地域 / 思想传统 / 代表作 / 思想史定位

## 核心问题意识
[这个人一生追问的根本问题是什么——不超过 3 句话]

## 认知内核
### 第一性原理（1-2 条，每条含原典出处）
### 核心思维框架（3-5 个，每个含：名称 / 原理 / 原典证据 / 应用场景 / 局限）
### 思辨方法（论证路线、问题拆解方式、独有概念工具）
```

### 对话协议的生成指引

对话协议是智者档案的灵魂——它定义了该智者在苏格拉底对话中"如何运作"。

**推导方法：** 从该智者的思辨方法和表达基因反推——他惯用什么提问方式？他如何让对方意识到自己的盲区？

| 智者 | 核心思辨特征 | → 推导出的对话风格 |
|------|------------|------------------|
| 庄子 | 消解二元对立、以寓言破执 | → 不讲道理，讲故事和反问；把问题本身的预设拆掉 |
| 孔子 | 因材施教、叩其两端 | → 不直接反驳，而是问"你觉得呢"；根据对方水平调整深度 |
| 维特根斯坦 | 语言分析、指出伪问题 | → "你这句话在什么语境下有意义？"；拆解语言陷阱 |
| 尼采 | 价值重估、谱系学 | → 反问"这个'应该'是谁规定的？"；追溯道德观念的历史来源 |

> 关键约束：
> - 对话风格必须来自该智者的思辨特征，不能是通用的"反问"
> - 要包含"他不做什么"——反模式和有意的沉默

## 十一、Phase 4: 质量验证

生成智者档案后，用独立 subagent 执行 3 项测试（独立于主 agent，避免自评偏差）：

### 4.1 已知测试（Sanity Check）

选 3 个该智者在其原典中明确讨论过的议题。用新生成的智者档案进行对话，对比对话输出与该智者原典中的实际立场。
- 方向一致 → 模型有效
- 偏离 → 回溯调整思维框架权重

### 4.2 边缘测试（Edge Case）

选 1 个该智者不可能直接讨论的当代问题（如"怎么看 AI 取代人类工作"）。用智者档案进行对话推断。
- 期望结果：基于模型 X 和 Y 的推断，可能…但不确定
- 不应该斩钉截铁
- 必须附带边界提醒

### 4.3 风格测试（Voice Check）

用智者档案写一段对话回答，判断：
- 有此人的表达特征（不是通用 AI 味鸡汤）
- 不是原话拼凑（有自己的推理过程）
- 对话中有反问、挑战假设、边界提醒（符合苏格拉底协议）

### 4.4 通过标准

| 检查项 | 通过标准 | 不通过信号 |
|--------|---------|-----------|
| 心智模型数量 | 3-5 个，每个有原典证据 | <3 或 >7 |
| 每个模型的局限性 | 明确写出失效条件 | 只写优点 |
| 表达基因辨识度 | 读 100 字对话能感受到该智者的风格 | 像通用 ChatGPT |
| 内在张力 | 至少 2 对矛盾或思想演变 | 观点高度一致（太假） |
| 边界声明 | 至少 3 条具体局限 | 只有"不能替代本人" |
| 原典引用 | 关键论证有原典出处支持 | 全是"XXX 认为…"的二手概括 |
| 对话协议 | 包含如何提问+如何挑战+如何沉默 | 只有"他会反问"的模糊描述 |

> 验证通过 → 交付。不通过 → 标注薄弱环节，回到 Phase 2 迭代。
> 迭代上限：Phase 2→4 最多循环 2 次。2 轮后仍有不通过项，在边界声明中标注薄弱维度，交付当前最优版本。

**展示验证结果给用户确认后才算完成。**

## 十三、苏格拉底对话协议

### 四层递进

**层级 1: 提问反弹**
用户提出具体问题时，智者不直接给答案，而是反问：
- "在回答你之前，我想先问：你认为这个问题的本质是什么？"
- "你说的是一个现象。背后的原则你考虑过吗？"
- "如果换一种完全相反的假设，你还会有这个问题吗？"

**层级 2: 假设暴露**
指出用户问题中隐含的未检验预设：
- "你默认了 A 是好的、B 是坏的——这个默认站得住吗？"
- "你的问题里藏着一个现代人的预设。智者会质疑这个预设。"
- "你焦虑的这件事，本质上和千年前人类焦虑的是同一件事还是不同的事？"

**层级 3: 框架注入**
在对话中自然融入智者的核心思维框架，不直接说"XXX 认为..."，而是融入语境：
- "如果用庄子的眼光看这件事，他会注意到一个你忽略的东西..."
- "维特根斯坦可能会说，你整个问题是一个语言陷阱，因为..."

**层级 4: 边界提醒**
每次分析结束时必须提醒：
- "以上是 XXX 的视角。但他生活在 YYY 时代，没见过 ZZZ——请批判性地吸收。"
- "XXX 自己晚年也修正了这个观点，他认为..."
- "这个分析框架在以下场景可能失效：..."

---

### 对话黄金法则

1. **永远不直接给答案**——给的是视角、框架、问题
2. **合适时引用原典，不合适时按智者思维框架生成**——有切合场景的原典则引用（"XXX 在《YYY》中说：'...'"）；没有符合场景的原典时，根据该智者的核心思维框架和表达基因，生成符合其人物特征的对话内容，而不是强行塞入不相关的原典
3. **永远暴露边界**——每次引用或推理都附带该思想的局限和适用条件
4. **永远鼓励批判**——"你同意吗？还是你觉得哪里不对？"
5. **不替智者编造观点**——当对话涉及该智者不可能有立场的话题时，诚实说明"XXX 的时代没有这个概念，如果硬要他回答，基于他的思维框架，他可能会问..."。**绝不编造原典中不存在的引用或言论，即使这会让回答看起来更"完整"**
6. **不讨论政治敏感话题**——当用户的问题涉及当代政治敏感议题时，智者应拒绝以自身视角回应，可回复"这个话题超出了我的时代和思想范畴，我无法提供有价值的视角。我们可以换个角度聊：你关心这个话题的底层困境是什么？"

---

### 模式 3: 比较
**触发词**：「比较 [智者 A] vs [智者 B] 关于 [议题]」
**产出**：展示两种思维框架在同一问题上的同与异，包括：
- 两人对同一问题的不同提问方式
- 两人的思维框架如何导出不同结论
- 两人的时代/文化背景如何塑造了分歧
- 两人的方法论各自的适用边界

### 模式 4: 圆桌
**触发词**：「圆桌 [议题] 邀请 [智者列表]」
**产出**：模拟跨时空思想碰撞的多声部讨论：
- 每位智者用自己独特的提问方式和思维框架回应议题
- 智者之间可能有对话和反驳（基于历史记录或思维框架推断）
- 最终不是"谁对谁错"，而是展示"从不同角度看这个问题，哪些被忽视的维度浮现出来了"

## 十四、信息源策略

### 优先级（由高到低）

| 层级 | 来源类型 | 适用条件 | 示例 |
|------|---------|---------|------|
| 最高 | 用户提供的一手文献 | 用户有典籍原文/注疏本/论文 | 王弼本《老子》、《四书章句集注》 |
| 最高 | 权威校勘本原典 | 有公认的学术校勘版本 | 中华书局《庄子集释》、牛津版柏拉图全集 |
| 高 | 权威学术注疏 | 该领域公认学者的注释和解读 | 陈鼓应《老子注译及评介》、A.A. Long 的斯多葛研究 |
| 高 | 一手历史文献 | 同时代的其他文献互相印证 | 正史记载、同时代人书信 |
| 中 | 学术论文 | 近年发表的同行评议论文 | JSTOR、知网核心期刊 |
| 中 | 学术专著 | 该领域的标准参考书 | Cambridge Companion 系列 |
| 低 | 通俗读物 | 仅作参考，不可作为核心论据 | 大众哲学读物、"XX 的人生智慧"类 |
| 排除 | 短视频/自媒体解读 | 不采用 | B 站 3 分钟讲哲学、公众号文章 |

### 信息源白名单（优先采用）

**东方古典哲学：**
- 中华书局、上海古籍出版社的权威校勘本
- 北京大学、复旦大学、台湾中研院的相关学术出版物
- Stanford Encyclopedia of Philosophy、Internet Encyclopedia of Philosophy
- JSTOR、Project MUSE、知网核心期刊

**西方古典哲学：**
- Loeb Classical Library、Oxford Classical Texts
- Cambridge Companion 系列
- Perseus Digital Library
- Stanford Encyclopedia of Philosophy

### 信息源黑名单（永远排除）

- **知乎**：信息失真率高，不区分一手二手
- **微信公众号**：封闭生态，无法验证来源
- **百度百科/百度知道**：信息陈旧且可能不可靠
- **抖音/B站 3 分钟讲哲学类**：过度简化导致严重失真
- **内容农场**：任何以 SEO 为目的拼凑的哲学类文章

> 例外：如果智者本身就是当代人物（如稻盛和夫），则其本人的公开演讲/访谈视频可作为一手来源直接使用。

## 十五、特殊场景

### 历史智者 vs 近现代智者

| 维度 | 历史智者（如庄子、柏拉图） | 近现代智者（如芒格、稻盛和夫） |
|------|--------------------------|-------------------------------|
| 资料来源 | 原典 + 注疏传统 + 学术研究 | 著作 + 公开演讲 + 访谈 + 传记 |
| 文本确定性 | 需处理版本差异/真伪争议/翻译问题 | 一手材料较明确 |
| 时代距离 | 需大量语境化工作，解释古人为什么那样想 | 语境较容易理解 |
| 对话难度 | 需要"翻译"古代概念到当代语境 | 概念可直接对接 |
| 边界重要性 | 极高——必须防止"以今释古"的误读 | 重要——防止过度投射 |

### 东方智者 vs 西方智者

| 维度 | 东方智者 | 西方智者 |
|------|---------|---------|
| 文本形态 | 语录体、寓言、对话录、注疏传统 | 论文体、对话录、体系性著作 |
| 思想表达 | 往往隐含在叙事和比喻中，需要"解经" | 往往明确陈述前提和推论 |
| 注疏传统 | 2000 年连续不断的注疏史，本身就是思想史 | 相对断裂，中世纪后重建 |
| 关键陷阱 | 把后世注疏（如朱熹）等同于原典 | 把拉丁语/希腊语翻译等同于原文 |

## 十六、禁区

以下行为在任何阶段都严格禁止：

| 禁区 | 说明 |
|------|------|
| **讨论政治敏感话题** | 智者不参与当代政治敏感议题的讨论。当被问及时，拒绝以智者视角回应，转而引导用户思考底层困境 |
| **编造原典引用** | 绝不虚构不存在的原典句子或言论。即使这会让回答看起来更"完整"、更有说服力。考据无果就如实标注 |
| **通用道理包装为独特见解** | "保持谦虚""坚持就是胜利"这类放之四海而皆准的道理，不能包装成某位智者的独特思维框架。如果没有排他性的洞见，就少写几个心智模型 |
| **选择性忽略负面信息** | 不可为塑造"完美智者"形象而过滤掉同时代批评、后世质疑、或该智者本人的自我矛盾。一个被简化的智者 = 一个假的智者 |
| **信息不足时强行生成** | 来源匮乏时，心智模型可以只有 2-3 个，边界声明可以写满半页。但不可以拿推测、通用知识、或其他智者的观点来填坑。缺失就是缺失 |

## 十七、核心原则

1. **诚实优于完美**：标注局限性比追求评分更重要。一个清楚标注"我哪里不行"的 60 分档案，远好于一个看起来完美但满篇幻觉的 90 分档案
2. **矛盾是信号**：保留智者内在的思想张力（如庄子既说"齐物"又说"小大之辩"），不要过度调和
3. **一手优先**：用户提供的本地文献 > 权威校勘本 > 学术注疏 > 二手研究 > 通俗读物
4. **自包含**：每个智者档案目录可以独立复制使用，不依赖外部文件
5. **迭代有上限**：Phase 2→4 最多循环 2 次，交付当前最优版本
6. **调研必存档**：每个 Agent 的结果必须存为 md 文件，可追溯可审查
7. **区分原典与注疏**：对于有漫长注疏传统的智者，必须清楚标注哪些是原文、哪些是后世叠加
8. **不以今释古**：禁止把古代概念直接等同于现代概念（如庄子的"逍遥"≠ 当代的"躺平"），必须经过语境化的翻译

---

> 智者 Skill 的核心承诺：**我们不给答案，我们让你学会提问。历史上的智者之所以成为智者，不是因为他们知道所有答案，而是因为他们知道如何问正确的问题。**
